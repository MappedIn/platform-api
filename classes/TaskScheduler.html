TaskScheduler Class

Private

Module: Mappedin

A class that handles scheduling tasks to be performed within the time constraints of frame updates, triggered using requestAnimationFrame.

Tasks can either be scheduled once per frame (called "frame updates"), or to execute as many times per frame as there is room for after processing all frame updates (called "frame tasks").

Posted tasks will always run during the same frame, assuming there's time to process them. However, an update or task that posts an update will not run that update until the next frame.

If the given task or group is in the queue, remove it. Cancelling a completed task is a no-op.

Canceling an individual task will also remove it from its task group, if it isn't already complete.

clear ()

Empty the task scheduler's queue. All tasks in it will be cancelled.

destroy ()

Destroy this task scheduler and remove all tasks from it.

flushTasks ()

Execute all tasks that are currently pending, without regard for the available frame time.

lastFrameTime () Number

Return the amount of time elapsed during the last completed frame update, in milliseconds. This value is unspecified if the task scheduler's update method has not yet run or if the scheduler has been cleared. If called during a frame update, this will return the time elapsed during the previous frame update.

Returns:

Number:

time elapsed during last frame update, in milliseconds

Set the provided task or group not to run, until it's resumed using resumeTask.

Tasks that are in a group can't be postponed individually; instead, call postponeTask with the entire group.

Parameters:

  • task FrameTask or FrameTaskGroup

    the task to postpone

requestAnimationFrame ()

Schedule this task scheduler to execute updates and tasks on the next available animation frame.

Allow a postponed task or task group to resume processing. This will also request an animation frame to allow the resumed task to be processed.
